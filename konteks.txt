Anda adalah senior full-stack engineer yang melanjutkan project `battleground-calc` dari state kode saat ini. File ini adalah living summary terbaru yang harus dipakai sebagai baseline kerja, tetapi hanya menyimpan konteks penting yang benar-benar relevan untuk patch berikutnya.

==================================================
RINGKASAN PROJECT
==================================================

Nama project:
- battleground-calc

Stack utama:
- Laragon
- CodeIgniter 4
- CodeIgniter Shield
- PHP 8.3+
- MariaDB/MySQL
- Bootstrap
- Dark mode only

Konsep aplikasi:
- admin score calculator untuk tournament battleground / FF / PUBG style
- 1 tournament memiliki banyak pot
- 1 pot memiliki banyak team
- 1 team memiliki banyak score per game
- total point = placement point + kill point
- leaderboard per pot

Modul utama:
- Dashboard
- TournamentController
- PotController
- TeamController
- ScoreController
- LeaderboardController
- ImportController
- Auth via Shield

Database / auth:
- jangan reset database
- jangan drop tabel/database
- jangan bongkar Shield tanpa alasan kuat
- utamakan patch kecil, aman, non-destruktif

==================================================
LOGIKA BISNIS YANG WAJIB DIPERTAHANKAN
==================================================

Status tournament:
- `belum_mulai`
- `start`
- `selesai`

Aturan:
- saat `selesai`, semua input/edit/add/delete harus read-only / terkunci
- saat `belum_mulai` atau `start`, score/pot/team masih boleh dikelola

Placement point:
- 1 => 12
- 2 => 9
- 3 => 8
- 4 => 7
- 5 => 6
- 6 => 5
- 7 => 4
- 8 => 3
- 9 => 2
- 10 => 1
- 11 => 0
- 12 => 0

PRINSIP IMPLEMENTASI
==================================================

Preferensi patch:
- konservatif
- kecil tapi efektif
- jangan ubah terlalu banyak file jika tidak perlu
- prioritaskan kestabilan tombol/logic daripada animasi atau styling berlebihan

Preferensi teknis:
- structural action tetap boleh submit biasa jika itu paling stabil
- AJAX dipakai untuk area yang memang cocok:
  - autosave score
  - autosave nama pot / urutan pot
  - sebagian aksi Team Manager
  - hapus team per-row di score table

Jangan lakukan:
- reset DB

- membuat semua action jadi AJAX kalau malah bikin regresi
- mengembalikan light mode

==================================================
ARAH UI/UX
==================================================

Prinsip UI:
- dark mode only
- compact / ramping / flat
- minim ruang kosong
- toolbar dan panel harus efisien
- jangan ada teks gelap di background gelap
- jangan ada elemen Bootstrap putih yang lolos

Halaman score:
- header atas cukup judul tournament + badge status + tombol penting
- quick summary tidak diperlukan
- langsung masuk ke pot modules

Tabel score:
- tidak boleh vertical scroll internal
- row bertambah ke bawah normal
- horizontal scroll hanya fallback terakhir
- 5 game harus aman, 6-8 game tetap rapi

Kolom Team:
- hanya nama team editable
- member tidak lagi ditampilkan di kolom Team
- jangan kembalikan hover member / hidden member UI ke kolom Team

Lookup team/player:
- sederhana, compact, native-ish
- lebih cocok datalist / combobox ringan
- jangan popup besar yang menutupi tabel
- format hasil konsisten:
  - `Nama Team, Player1, Player2, Player3, ...`
