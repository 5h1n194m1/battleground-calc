==================================================
MODEL DOMAIN DAN RELASI BISNIS
==================================================

Konsep utama:
- 1 tournament memiliki banyak pot/grup.
- 1 pot memiliki banyak team.
- 1 team memiliki banyak member/player.
- 1 team memiliki banyak score per game pada sebuah pot.
- Total point = placement point + kill point.
- Leaderboard dihitung per pot dari total score semua game.

Status tournament:
- `belum_mulai`
- `start`
- `selesai`

Label UI status:
- `belum_mulai` -> `Standby`
- `start` -> `Live`
- `selesai` -> `Finished`

Aturan lock/read-only:
- Jika status tournament `selesai`, semua input/edit/add/delete terkait tournament harus terkunci/read-only.
- Jika status `belum_mulai` atau `start`, pengelolaan pot/team/score masih boleh dilakukan.
- Lock harus ada server-side, bukan hanya disabled button di view.
- Fungsi terkait:
  - `TournamentModel` status constants dan `statusOptions()`
  - `PotController::isTournamentEditable()`
  - `TeamController::isPotEditable()`
  - `ScoreController::isEditableStatus()`

Placement point wajib dipertahankan:
- Rank 1  -> 12
- Rank 2  -> 9
- Rank 3  -> 8
- Rank 4  -> 7
- Rank 5  -> 6
- Rank 6  -> 5
- Rank 7  -> 4
- Rank 8  -> 3
- Rank 9  -> 2
- Rank 10 -> 1
- Rank 11 -> 0
- Rank 12 -> 0

Batas score:
- `ScoreController::MAX_RANK = 12`.
- Rank valid 1 sampai 12.
- Kill point minimal 0.
- Pada `saveBulk`, rank dan kill harus diisi berpasangan untuk game yang dipakai.
- Rank/kill kosong pada sebuah game berarti game itu tidak disimpan untuk row tersebut.

Jika mengubah placement map, wajib sinkronkan:
- `ScoreController::placementMap()`
- data placement map yang dikirim ke `scores/_calculator.php`
- logic total di `public/assets/js/score-workspace.js`
- trigger DB jika database aktif punya trigger score
- dokumen konteks ini

Catatan tipe kolom penting:
- Migration awal memakai `BIGINT UNSIGNED` untuk `tournaments`, `pots`, `teams`, dan `scores`.
- Beberapa migration lanjutan memakai `INT UNSIGNED` untuk `team_members.team_id`, `teams.pot_id`, dan `teams.tournament_id`.
- Karena ada riwayat patch manual DB, jangan asumsi tipe di database aktif tanpa cek schema lokal.
- Sebelum migration yang menyentuh FK, cek tipe aktual dengan `DESCRIBE`/information_schema agar tidak muncul mismatch FK.

Tabel `tournaments`:
- `id` unsigned PK auto increment
- `name` varchar(150)
- `status` varchar(20), default bisnis `belum_mulai`
- `created_at`, `updated_at`

Tabel `pots`:
- `id` unsigned PK auto increment
- `tournament_id` unsigned FK tournaments
- `name` varchar(150)
- `sort_order` int unsigned default 0
- `reference_image_1` varchar(255) nullable
- `reference_image_2` varchar(255) nullable
- `created_at`, `updated_at`

Tabel `teams`:
- `id` unsigned PK auto increment
- `tournament_id` unsigned nullable FK tournaments, ON DELETE SET NULL
- `pot_id` unsigned nullable FK pots, ON DELETE SET NULL
- `name` varchar(150)
- `sort_order` int unsigned default 0
- `created_at`, `updated_at`
- `pot_id = NULL` berarti team belum masuk pot.
- `tournament_id` tetap penting untuk team yang belum punya pot.

Tabel `team_members`:
- `id` unsigned PK auto increment
- `team_id` unsigned FK teams, ON DELETE CASCADE
- `player_name` varchar(150)
- `player_role` varchar(100) nullable
- `created_at`, `updated_at`

Tabel `scores`:
- `id` unsigned PK auto increment
- `pot_id` unsigned FK pots, ON DELETE CASCADE
- `team_id` unsigned FK teams, ON DELETE CASCADE
- `game_no` int unsigned
- `rank_no` int unsigned
- `kill_point` int unsigned default 0
- `placement_point` int unsigned default 0
- `total_point` int unsigned default 0
- unique key `(pot_id, team_id, game_no)` dengan nama `scores_pot_team_game_unique`
- index `pot_id`, `team_id`

`ScoreController`:
- `index(potId)`: halaman score workspace untuk seluruh pot dalam tournament yang sama.
- Load semua pot tournament, semua team dari pot tersebut, member, score, totals, max game, dan pot modules.
- Saat render, team tiap pot diurutkan berdasarkan total score desc, lalu `sort_order`, lalu nama.
- `saveBulk()`: endpoint utama autosave/submit score workspace.
- `saveBulk()` bisa:
  - update metadata pot (`pot_name`, `pot_sort_order`)
  - update nama team
  - create team baru
  - attach team tanpa pot ke pot
  - sync member dari textarea
  - validasi duplicate nama team dalam scope tournament
  - replace score matrix untuk pot tersebut
- `save()`: endpoint score single/simple lama.
- `placementMap()`: mapping placement point.
- `duplicateScopeTeams()` dan `duplicateNameMessage()`: duplicate guard.

Catatan penting `saveBulk()`:
- `saveBulk` bersifat replace-state untuk pot terkait.
- Ia delete semua score lama pada `pot_id` tersebut untuk `game_no <= game_count`, dan juga delete `game_no > game_count`, lalu insert ulang payload baru.
- Jangan panggil `saveBulk` untuk update satu cell kecuali request mengirim matrix lengkap yang masih ingin dipertahankan.
- AJAX response memberi `reloadPage=true` jika row/team temporary baru perlu sinkron dengan ID DB.

`ImportController`:
- `teams()`: halaman import team.
- `storeTeams()`: upload dan proses import team.
- Format didukung: CSV, TXT/TSV, XLSX.
- Parser XLSX manual via `ZipArchive` dan XML.
- Guard security import:
  - max 500 rows per file
  - max upload 4096 KB
  - max XML chunk penting 4 MB
  - MIME/extension dibatasi

`LeaderboardController`:
- `pot(potId)`: leaderboard readonly per pot.
- Query aggregate total score, kill, placement, games played dari team yang attached ke pot.
- Sorting saat audit: `total_score DESC`, lalu `teams.name ASC`.

SECURITY DAN CSP
==================================================

Filter aktif:
- `csrf` alias ke `App\Filters\GracefulCsrfFilter`
- `session` Shield `SessionAuth`
- `group` Shield `GroupFilter`
- `idle` custom `IdleTimeoutFilter`
- `localonly` custom `LocalOnlyFilter`
- `auththrottle` custom `AuthThrottleFilter`
- `appheaders` custom `AppSecurityHeadersFilter`

Required filters:
- before: `localonly`, `invalidchars`
- after: `appheaders`

Global filters:
- before: `csrf`
- after: kosong

LocalOnly:
- `LocalOnlyFilter` hanya mengizinkan host `localhost` dan `127.0.0.1`.
- Jika aplikasi ingin dibuka dari device lain/LAN/hosting, filter `localonly` harus disesuaikan atau dilepas dari required filters.

CSRF:
- CSRF memakai session protection.
- Token name di config: `csrf_test_name`.
- Layout menaruh token aktif di dataset body:
  - `data-bg-csrf-token-name`
  - `data-bg-csrf-hash`
- Semua form POST harus disinkronkan via `BGApp.syncAllPostForms()`.
- Endpoint AJAX sebaiknya selalu mengembalikan `csrfTokenName` dan `csrfHash`.

CSP:
- Layout auth masih memakai nonce tag.
- Main app shell JS besar dipindah ke `public/assets/js/app-shell.js` agar lebih ramah CSP.
- Hindari inline JS baru kalau tidak perlu.

Upload:
- Reference image pot disimpan di `public/uploads`.
- `.htaccess` dan `index.html` sudah ada.
- Jangan longgarkan MIME/extension upload.

- Duplicate nama team dijaga per tournament.
- Jangan ubah menjadi global lintas semua tournament tanpa kebutuhan jelas.

7. Finished tournament lock
- Semua mutasi harus guard server-side.
- Jangan hanya mengandalkan disabled button.

8. Dark mode consistency
- Bootstrap default putih bisa bocor jika menambah komponen baru tanpa class/theme.
- Jangan ada teks gelap di background gelap.

Project sudah memiliki fondasi CRUD tournament/pot/team, roster global, import team, score workspace multi-pot, leaderboard, auth Shield, dan hardening security lokal. Modul paling sensitif adalah:
