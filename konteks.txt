Anda adalah senior full-stack engineer yang melanjutkan project `battleground-calc` dari state kode saat ini. File ini adalah living summary terbaru yang harus dipakai sebagai baseline kerja, tetapi hanya menyimpan konteks penting yang benar-benar relevan untuk patch berikutnya.

==================================================
RINGKASAN PROJECT
==================================================

Nama project:
- battleground-calc

Stack utama:
- Laragon
- CodeIgniter 4
- CodeIgniter Shield
- PHP 8.3+
- MariaDB/MySQL
- Bootstrap
- Dark mode only

Konsep aplikasi:
- admin score calculator untuk tournament battleground / FF / PUBG style
- 1 tournament memiliki banyak pot
- 1 pot memiliki banyak team
- 1 team memiliki banyak score per game
- total point = placement point + kill point
- leaderboard per pot

Modul utama:
- Dashboard
- TournamentController
- PotController
- TeamController
- ScoreController
- LeaderboardController
- ImportController
- Auth via Shield

Database / auth:
- jangan reset database
- jangan drop tabel/database
- jangan bongkar Shield tanpa alasan kuat
- utamakan patch kecil, aman, non-destruktif

Update 2026-04-24 - security hardening gabungan dari patch terbaru:
- Patch terbaru dari pasted text digabung ke project aktif tanpa mengubah logic bisnis Team Manager, score workspace, roster, atau import tournament context.
- Global `pagecache` di `app/Config/Filters.php` dihapus dari required filters karena berisiko menyajikan form lama dan token CSRF basi pada aplikasi login/stateful.
- CSRF sekarang memakai `App\Filters\GracefulCsrfFilter` agar kegagalan token diarahkan ke pesan login/expired yang lebih halus, termasuk response JSON untuk AJAX.
- Ditambahkan filter custom:
  - `app/Filters/LocalOnlyFilter.php`
  - `app/Filters/AppSecurityHeadersFilter.php`
  - `app/Filters/AuthThrottleFilter.php`
  - `app/Filters/GracefulCsrfFilter.php`
- Config security/session/CSP diperkeras:
  - `app/Config/Security.php`
  - `app/Config/Session.php`
  - `app/Config/ContentSecurityPolicy.php`
  - `app/Config/App.php`
  - `.env`
- `app/Views/layouts/main.php` tidak lagi menyimpan shell JS inline besar; shell dipindah ke `public/assets/js/app-shell.js` agar lebih ramah CSP.
- Semua form POST tetap disinkronkan CSRF oleh `window.BGApp.syncAllPostForms()` dan confirm inline diganti ke `data-confirm` / `data-confirm-click`.
- Upload screenshot pot diperkeras dengan `public/uploads/.htaccess`, `public/uploads/index.html`, dan `PotController::ensureUploadDirectory()`.
- Import team dibatasi dan diperkeras:
  - maksimal 500 baris per file
  - maksimal XML XLSX 4 MB per bagian penting
  - validasi MIME/ekstensi/ukuran file lebih ketat
  - parsing XML memakai `LIBXML_NONET | LIBXML_NOERROR | LIBXML_NOWARNING`
  - panjang nama team/player/contact/notes dipotong agar tidak membuat data abnormal masuk database
- Auth Shield diperketat:
  - registration disabled
  - magic link disabled
  - minimum password length 12
- Catatan lokal penting:
  - `LocalOnlyFilter` hanya mengizinkan host `localhost` / `127.0.0.1` dan IP lokal.
  - Jika aplikasi ingin dibuka dari device lain/LAN/hosting, filter `localonly` harus disesuaikan atau dilepas dari required filters.
